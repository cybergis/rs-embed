🛰️ rs-embed

Any Model · Any Place · Any Time. Remote-sensing foundation-model embeddings, on demand.

Eighteen state-of-the-art satellite foundation models, one interface. Give it where, when, and which model — get an embedding back. This demo tells the story through Riverside citrus, the crop that built this region.

Why it matters

Every satellite foundation model ships its own incompatible pipeline. Just comparing a few of them turns into weeks of brittle glue code — and the bugs are silent.

😖 The old way — per model

  • Authenticate & query Google Earth Engine
  • Pick the right bands, in the right order
  • Apply the model's exact normalization
  • Download checkpoints, write the forward pass
  • Repeat for every model · hope nothing is mis-wired
≈ weeks per model · silent correctness bugs

✅ With rs-embed

  • One unified ROI → embedding call
  • Swap models by changing a single string
  • Band order & normalization encoded once, correctly
  • Fair benchmarking is the default, not extra work
  • Reproducible exports with an audited manifest
1 line per model · fair, reproducible

The demonstration — four interactive scenarios

A Riverside agronomist (no ML background) drives the whole thing from a browser. Each scenario is one interactive module in the live notebook.

🌍 any place

Earth's Twins

Click anywhere on Earth and rs-embed finds its look-alikes worldwide by embedding similarity. A Riverside grove's twins? The world's other citrus belts.

You do: click a point → see its 5 global twins light up.
⏳ any time

Satellite Time Machine

Slide through 2017→2024. The embedding "change curve" spikes exactly when the ground changed — no labels, no change-detection model. Local pick: the shrinking Salton Sea.

You do: drag the year slider → watch the place transform.
🎨 any model

Instant Land-Cover

Embeddings are features. Cluster a region's grid and citrus, urban, and desert fall out with no labels. Swap the model and watch the map sharpen.

You do: draw a box, pick a model → instant segmentation.
🏆 fair comparison

Model Showdown

The payoff: which model maps citrus best? Imagery is fetched once and shared across models, every model scored on the same split — so only the model differs.

You do: pick models → a fair, ranked leaderboard.

Who it's for

rs-embed turns the hard 90% — imagery + per-model preprocessing — into a function call, so two very different people both get unblocked.

🌾 The domain scientist

An agronomist, ecologist, or economist who isn't a deep-learning engineer. They get state-of-the-art satellite features without touching Earth Engine quirks, band orders, or checkpoints — and spend their time on their question.

🔬 The ML researcher

Wants to know which foundation model wins on a task. The shared-input export_batch harness makes that comparison fair and reproducible by default, not by careful manual discipline.

Try it in your browser

The notebook runs cached & offline by default (booth-safe — no Earth Engine auth, no GPU). Flip RUN_LIVE = True to compute against live imagery.
